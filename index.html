選擇轉職的原因 : 努力虛擲、成長停滯

曾經我也勸過自己往好處看，莫再多想，生活無非圖個安穩；但日復一日，為重複空洞的工作忙碌加班， 為反覆出現的相同問題殫精竭慮，除了疲勞與內耗，實難感受到任何成長，同時我也無法自欺欺人， 以現職的光明面強壓心中漸趨高漲的滯塞感與危機感；最終，經過深刻自我審視與多方諮詢後， 下定轉職決心。

選擇軟體工程師的理由 : 實現自我、回饋社會

承上，幾經反思與探索，我總結出理想的職涯必須滿足 : 技術為本、不斷精進、回饋社會等三個條件； 一者，技術導向的工作較有客觀標準，有利自我衡量與持續精進，而打磨技術、累積經驗的過程， 除能厚實職涯的底氣，也可進一步完成自我提升與實現。

二者，若此生努力僅是為了獨善其身，難免有些可惜，故也希望於職涯中積累， 終能回饋社群、協助他人成長，也算實現進入公職時的初衷；綜合以上要素， 我認為軟體工程師最能契合需求，且數年前曾利用就職前的空檔， 參加臺大資訊系統訓練班「Python程式設計與網站擷取」課程，在職期間也負責過單位內部網頁的管理， 故對於撰寫程式、操作網頁並非全然陌生。

描述一件產生明顯負面情緒的經歷，如何處理該情緒？

工作起初，曾因不諳法令及一連串個人疏忽導致專案進度嚴重落後，除連累前輩、同事遭受責難，假日還須一同加班補救，當時的自責、懊惱與羞愧仍記憶猶新， 且因個性較不喜歡麻煩他人，故遭逢此類事件總是尤為難受，也因而衍生以下通案做法:

分析問題肇因，避免類案再生

處理情緒之前，還得先釐清肇生負面情緒的根源，否則有些治標不治本；而經由理性檢討分析後，就外在層面，能夠發掘問題癥結、擬定改進措施，防止造成負面心緒的事件重演， 就內在層面，檢視問題的過程，能暫時將注意力聚焦事件本身，以較客觀的角度看待問題，通常也能讓心情稍加平復。

表達真實感受，接納必然情緒

呈上，在理清問題及思緒後，我會主動向受到牽連的人致歉，目的並非尋求諒解，畢竟他人本無義務給予原諒，但真誠表達內心歉疚，不僅是與人互動的基本素養，也是與情緒達成和解的正向途徑；正如前述，世無完人，犯錯在所難免，只能要求自己以理性誠實的態度面對過錯，盡力彌補，而過程中產生的諸多情緒，無可避免也無須避免；是故，處理負面情緒，順其自然即可，意即不應刻意忽視、也不過份壓抑，允許其存在，並靜觀其生滅， 正如「西藏生死書」所提，雜念與情緒可將其比做浮雲，內心自性則如天空，天晴有時、陰霾亦有時，皆是自然，但晴雨之後，天空仍是天空，不曾或變。

關於這份申請網頁，分享一個開發時的技術心得

由於轉職目標是Java後端工程師，故起初學習完HTML、CSS及基礎Javascript後就未再複習，故製作本申請網頁時有些難以著手，但過程還是十分有趣，也重拾了一些初學時的記憶，惟由於網頁大多以基礎HTML、CSS實現，故僅分享以JavaScript實現輪播圖的技術心得 :

1. 初始化

在初始狀態下，為「個人介紹」的內容和對應按鈕設置 active 和 active-btn 類別，其中：

  • active 類別使內容的 display 屬性設為 block，讓該內容顯示在頁面上。
  • 其他內容的 display 屬性為 none，使他們處於隱藏狀態。

2. 選取內容和按鈕

使用 document.querySelectorAll 分別取得 .carousel-content（輪播內容）和 .nav-btn（導航按鈕）的 NodeLists。每個按鈕都會綁定一個 click 事件，當按鈕被點擊時會觸發 showContent 函數。

3. click 事件的工作原理

每次點擊按鈕時，click 事件會將該按鈕在 NodeLists 中的 index 作為參數傳給 showContent 函數。

4. showContent 函數的執行步驟

showContent 函數的執行步驟如下：

  • 遍歷所有內容和按鈕，檢查並移除其中的 active 和 active-btn 類別。
  • 這樣可以將所有內容和按鈕的狀態重置為初始狀態，確保後續僅有選中的內容和按鈕處於「顯示」和「被選中」狀態。

5. 顯示選中內容

根據選中按鈕的 index，在 NodeLists 中找到對應的內容和按鈕，並分別為它們添加 active 和 active-btn 類別。active 類別讓內容顯示在頁面上，而 active-btn 類別則用來標記當前選中的按鈕。

另本次開發過程中，對於許多CSS及Javascript語法已有些模糊，故通常在幾次嘗試未果後，便會轉向尋求ChatGPT等AI工具協助，通常可以較快取得理想回應， 但也深刻感受到，如果本身不具備相關知識，首先很難構思直擊痛點的問題，再者無法判定其回答是否切合需求，只能不斷重複試錯及提問的循環，反而徒增成本； 因此，製作本申請網頁時，仍舊花了一些時間回顧當初的學習資源，而經由再次釐清基本原理後也確實讓後續開發過程順利不少，顯見即使AI工具如何強大，若無堅實基礎仍舊無法發揮其功效。

如何看待自身工作和整個社會群體的連結關係？

過去我對於工作與社會連結的想法有些過於狹隘，總認為須直接有助益於社會方能稱之，但在經歷公職並接觸軟體學習後，想法已有轉變，具體如下 :

在工作層面 : 雖尚不清楚實際情形如何，但我認為軟體技術的目的是將使用者的需求轉化為具體可執行的工具或服務，工程師作為技術使用者，一來透由本職工作提供消費大眾更快、更穩定、更人性化的服務，從早期的社群通訊軟體、goole map到近年的外送平台、數位銀行等服務，均大幅提升了便利性並改變了人們的生活樣貌 ；二來，即使大多工程師本身可能並未參與技術工具的開發，但使用工具的同時，可透由疊加自身經驗創造更多可能性， 也能適時給予反饋，促進技術迭代，從而使軟體技術生態系更加豐富完善，不僅有助軟體社群共同成長，技術進步也能打造更好的應用，創造或滿足更多社會需求。

在個人層面 : 自學過程中受惠於諸多技術社群或個人的分享，對於軟體社群的活躍與開放感到非常認同，於是希望自己在累積足夠經驗與技術後，也能夠做為資訊的提供者，為他人解惑，為軟體社群略盡綿薄，回饋在成長學習過程中收穫的幫助與善意，促進正向循環。再者，雖目前尚無法構想，但掌握一門技術後，將其用於有利公眾之事，也是我對職涯的期許， 同前所述，軟體做為滿足現代社會需求的主要手段之一，相信仍有著無限可能等待發掘，期待未來具備足夠能力後，能透由自身技術力造福人群，也算回應當初選擇公職的初衷。
